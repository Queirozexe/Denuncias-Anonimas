# 🚨 Sistema de Denúncias Anônimas

Projeto acadêmico desenvolvido com foco em **design, usabilidade e organização de informações**, utilizando apenas tecnologias web estáticas (HTML, CSS e JavaScript) para publicação via GitHub Pages.

---

## 🎯 Objetivo

Permitir que cidadãos realizem denúncias de forma:

* ✅ Anônima (sem necessidade de identificação)
* ✅ Organizada (com categorias estruturadas)
* ✅ Responsável (com descrição detalhada)
* ✅ Acompanhável (via protocolo gerado automaticamente)

---

## 🧩 Funcionalidades

### 📌 1. Categorias de denúncia

Organizadas em blocos:

* **Segurança e Violência**
* **Direitos e Conduta**
* **Meio Ambiente e Bem-Estar**
* **Cidade e Serviços**

---

### 📍 2. Localização

Campos disponíveis:

* Bairro
* Rua
* Ponto de referência

---

### 📎 3. Upload de evidências (simulado)

* Imagem
* Vídeo
* Documento

> ⚠️ Obs: funcionamento apenas visual (não envia para servidor)

---

### 🔢 4. Protocolo automático

Cada denúncia gera um código único:

Exemplo:

```
PROTO-240401-1234
```

---

### 🔍 5. Acompanhamento de status

Consulta por protocolo com etapas:

* Denúncia recebida
* Em análise
* Encaminhada
* Concluída

---

### 🛠️ 6. Painel Administrativo

Permite gerenciar denúncias:

* 📋 Listagem completa
* ✏️ Edição de dados
* 🔄 Alteração de status
* ⏩ Avançar status rapidamente
* ❌ Exclusão de denúncias
* 🔎 Filtro por texto

---

## 💾 Armazenamento

Este projeto utiliza:

```
localStorage (armazenamento local do navegador)
```

Ou seja:

* Não há backend
* Não há banco de dados real
* Os dados ficam salvos apenas no navegador

---

## 🧪 Natureza do projeto

Este sistema é uma **simulação acadêmica**, ideal para:

* Exercícios de front-end
* Demonstração de interface
* Protótipos funcionais

---

## 🖥️ Tecnologias utilizadas

* HTML5
* CSS3 (design moderno + responsivo)
* JavaScript (lógica e armazenamento local)

---

## ⚠️ Observações finais

* Este projeto **não deve ser usado em produção**
* Não há segurança real de dados
* Não possui backend ou autenticação
